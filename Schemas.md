
### 1. Campaign
```python

class Campaign(Base):

  

"""

SQLAlchemy model representing a campaign.

  

This model defines the structure of the `campaign` table and stores

information about individual campaigns, including their associated

dungeon master (DM).

  

Attributes:

id (int): Primary key identifier for the campaign.

name (Optional[str]): Name of the campaign (max 100 characters).

description (Optional[str]): Description of the campaign (max 500 characters).

dm_id (int): Foreign key referencing the user who is the DM.

createad_at (Optional[datetime]): Timestamp of campaign creation.

dm (User): The user who owns and manages the campaign.

member (List[CampaignMember]) List of campaings with user.

"""

  

__tablename__ = "campaign"

  

id: Mapped[int] = mapped_column(primary_key=True, index=True)

name: Mapped[Optional[str]] = mapped_column(String(100))

description: Mapped[Optional[str]] = mapped_column(String(500))

dm_id: Mapped[int] = mapped_column(ForeignKey("users.id"))

created_at: Mapped[Optional[datetime]] = mapped_column(DateTime(timezone=True))


  

dm: Mapped["User"] = relationship(back_populates="campaign")

members: Mapped[List["CampaignMember"]] = relationship(back_populates="campaign")

campaign_npc: Mapped[List["NPC"]] = relationship(back_populates="campaign")

characters: Mapped[List["Character"]] = relationship(back_populates="campaign")

token: Mapped[List["Token"]] = relationship(back_populates="campaign")

attle_sessions: Mapped[List["BattleMapSession"]] = relationship(back_populates="campaign")

```

Za  "Campaign" potrebno je definirati:

*Campaign je ode samo placeholder vridi za sve*

- CampaignBase (BaseModel)
	- Nasljeđuje iz "Base model" ove su djeljene za Create, Update i Read
- CampaignCreate (CampaignBase)
	- Nasljeđuje iz "Campaign Base"
- CampaignUpdate (BaseModel)
	-  Updatea opcionalne stvari
- CampaignRead (CampaignBase)
	- Vraća podatke

1. Is it generated by the DATABASE?     → Read only        (id, created_at)
2. Is it required to CREATE a record?   → Create only      (foreign keys like user_id)
3. Is it NEVER returned in responses?   → Create only      (password)
4. Can it CHANGE after creation?        → Base + Update    (name, description)
5. Is it SHARED across all operations?  → Base             (name, description)

```python
class CampaignBase(BaseModel):

  

name: Optional[List] = None

description: Optional[List] = None

  

class CampaignCreate(CampaignBase):

  

dm_id: int

  

class CampaignUpdate(BaseModel):

  

name: Optional[List] = None

description: Optional[List] = None

  

class CampaignRead(CampaignBase):

  

id: int

dm_id: int

created_at: Optional[List] = None
```


### 2. Korištenje kasnije

Schemas are used in **3 main places** in a FastAPI project:

---

## 1. Route Endpoints — Validating Incoming Data

```python
@router.post("/campaigns")
def create_campaign(campaign: CampaignCreate, db: Session = Depends(get_db)):
    #                ^^^^^^^^^^^^^^^^
    #                FastAPI automatically validates the request body
    #                against CampaignCreate schema
```

If someone sends invalid data (wrong type, missing field) FastAPI **automatically rejects it** with a 422 error — you don't write any validation yourself.

---

## 2. Route Endpoints — Shaping Outgoing Data

```python
@router.get("/campaigns/{id}", response_model=CampaignRead)
def get_campaign(id: int, db: Session = Depends(get_db)):
    #                      ^^^^^^^^^^^^^^^^^^^^
    #                      FastAPI filters the response through CampaignRead
    #                      so sensitive fields like password never leak out
    campaign = db.query(Campaign).filter(Campaign.id == id).first()
    return campaign
```

---

## 3. CRUD Functions — Moving Data Between API and DB

```python
def create_campaign(db: Session, data: CampaignCreate) -> CampaignRead:
    #                             ^^^^^^^^^^^^^^^^          ^^^^^^^^^^^^
    #                             what comes in             what goes out

    new_campaign = Campaign(**data.model_dump())
    db.add(new_campaign)
    db.commit()
    db.refresh(new_campaign)
    return new_campaign
```

---

## The Full Flow

```
HTTP Request
    │
    ▼
CampaignCreate         ← validates & parses incoming JSON
    │
    ▼
CRUD function          ← creates SQLAlchemy model from schema
    │
    ▼
Database               ← saves the record
    │
    ▼
CampaignRead           ← shapes what gets returned
    │
    ▼
HTTP Response          ← clean JSON back to the client
```

---

## Summary

|Schema|Used for|
|---|---|
|`Create`|Validating data coming IN to create a record|
|`Update`|Validating data coming IN to update a record|
|`Read`|Shaping data going OUT in responses|
|`Base`|Never used directly — just a parent for the others|

Essentially schemas are the **gatekeepers** of your API — nothing gets in or out without passing through them.
